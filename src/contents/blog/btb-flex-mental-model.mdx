Le développement web est devenu l'un des domaines les plus dynamiques et recherchés dans l'industrie informatique. Pour devenir un développeur web compétent, il est essentiel de maîtriser les langages de base, notamment HTML et JavaScript. Mais se pose la question cruciale : faut-il réellement apprendre d'abord le HTML avant de plonger dans le JavaScript ? Dans cet article, nous explorerons les arguments en faveur et en défaveur de chaque approche pour aider les apprenants à faire le choix qui leur convient le mieux.

## Ce qu'il y a réellement derrière le HTML

Le HTML (HyperText Markup Language) est la pierre angulaire du développement web. Il définit la structure de base d'une page web en utilisant des balises pour décrire le contenu et son organisation. Mais est-il autant nécessiteux de se lancer dans son apprentissage ? Avant de vous donner mon avis, autant vous dire pourquoi l'on se lance dans son apprentissage :

### La simplicité du langage

Le HTML est un langage relativement simple et facile à apprendre. En commençant par ce langage, les débutants peuvent se familiariser avec les bases de la création de pages web avant de passer à des concepts plus avancés.

### Structuration logique

Comprendre le HTML permet d'acquérir une pensée structurée pour organiser le contenu d'une page web de manière logique et sémantique. Cela favorise une meilleure compréhension des autres langages et frameworks utilisés en développement web.

### Accessibilité

Le HTML est la base de l'accessibilité web. En maîtrisant les bonnes pratiques dès le début, les développeurs peuvent créer des sites web accessibles à un large public, y compris aux personnes handicapées ou défavorisées par le numérique.

## La team JavaScript : Efficacité et réalisation rapide

Le JavaScript est un langage de programmation côté client, permettant d'ajouter des fonctionnalités interactives et dynamiques aux pages web. Et voici les raisons pour lequelles vous devriez obligatoirement vous lancer, selon moi :

### Augmentation de sa motivation

Beaucoup d'apprenants sont motivés par le résultat rapide de l'apprentissage du JavaScript. En créant des interactions et des animations dès le début, ils peuvent constater rapidement les fruits de leur travail, ce qui peut les encourager à continuer à apprendre. C'est la raison pour laquelle je vous conseille énormement le Javascript. Vous serez toujours plus passionné par le web, et sans relâche.

### Forte polyvalence

Le JavaScript est utilisé non seulement pour le développement web mais aussi pour les applications côté serveur, les applications mobiles, etc. En commençant par JavaScript, les apprenants peuvent explorer différents domaines de développement dès le début.

### Forte évolution des méthodes

Le développement web moderne met l'accent sur les applications web interactives. En commençant par JavaScript, les développeurs se plongent directement dans l'environnement actuel du web.

## Mon avis sur ses deux langages classiques

Alors, doit-on apprendre d'abord le HTML avant de faire du JavaScript ? Ou peut-on directement commencer par le JavaScript ? En fin de compte, le choix dépend des préférences personnelles, du style d'apprentissage et des objectifs de chaque individu. Commencer par le HTML peut fournir une base solide en structuration et en sémantique, tandis que plonger directement dans le JavaScript peut inspirer la créativité et l'interactivité. Une approche intermédiaire combinant les deux langages peut offrir une compréhension globale et pragmatique du développement web.

Mais ayant été débutant, je n'ai pas passé énormement du temps sur du HTML. Comparé à la puissance du JavaScript et notamment du framework VueJS ou de d'autres frameworks, passer du temps pour découvrir des balises, s'y professionaliser. Ne prenez pas plus d'une semaine sur le HTML si vos ambitions sont de devenir un très bon crack en développement frontal. Maitrisez, oui, mais sans pendre du temps.
